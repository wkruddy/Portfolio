# Portfolio research — iteration loop

Research is **not** a one-off web search. Each run should improve how the next run finds, scores, and extracts signal for Kyle’s Staff++ portfolio.

## Loop

```
Plan queries → Search/browse → Score references → Extract patterns → Update learnings.md → Refine next-run strategy
```

| Step | Output |
|------|--------|
| **Plan queries** | 3–6 targeted queries + which sources to revisit (`sources.md`) |
| **Search/browse** | WebSearch for discovery; re-open curated URLs for depth |
| **Score references** | Rubric in `portfolio-research` skill (5 criteria) |
| **Extract patterns** | IA, hero proof, work proof, motion density, contact friction |
| **Update learnings.md** | Append run log + revise methodology if tactics changed |
| **Refine strategy** | Note what to try, avoid, or add to `sources.md` next time |

## Files

| File | Role |
|------|------|
| [learnings.md](learnings.md) | Living methodology + **run log** (append every research session) |
| [sources.md](sources.md) | Curated URLs and collections — prefer revisiting over random search |
| [../handoffs/research-to-ux.md](../handoffs/research-to-ux.md) | Design Brief for UX (per initiative) |

## Agent contract

1. Read [SOUL.md](../../SOUL.md) and [learnings.md](learnings.md) **before** researching.
2. Follow [.cursor/skills/portfolio-research/SKILL.md](../../.cursor/skills/portfolio-research/SKILL.md).
3. **After** each run: append structured entry to `learnings.md` (date, queries, worked/failed, next iteration).
4. In `research-to-ux.md`, cite **learnings version** (date of last run-log entry or “methodology vN” in learnings header).

## When to skip research

Small copy tweaks or bugfixes that UX can spec from codebase alone — orchestrator may waive research. New hero/work positioning or IA changes should run the loop.

## Human (Kyle) in the loop

- Approve positioning tensions before UX locks spec.
- Add/remove URLs in `sources.md` when you find a reference worth re-scoring.
- Answer `[confirm with Kyle]` items in briefs — never let agents guess employers or metrics.
